Reporte de Ingeniera: Prediccin de Rendimiento Acadmico

1. Introduccin y Definicin del Problema

En el ecosistema educativo contemporneo, el rendimiento acadmico es un fenmeno multifactorial influenciado por variables socioeconmicas, hbitos de estudio y antecedentes escolares. La capacidad de anticipar la calificacin final de un estudiante ($G3$) no es solo un ejercicio estadstico, sino una herramienta estratgica para la deteccin temprana de riesgos y la personalizacin del aprendizaje.

1.1 Objetivo

Desarrollar un modelo de regresin lineal robusto capaz de predecir la nota final de estudiantes de nivel secundaria/preparatoria basndose en datos demogrficos y acadmicos previos.

1.2 Filosofa de Diseo: Parsimonia

Este anlisis se rige por el Principio de Parsimonia (Navaja de Ockham). En ingeniera de datos, la complejidad conlleva costos computacionales y riesgos de sobreajuste. Por ello, nuestra mtrica de xito no es solo la precisin pura ($R^2$), sino la eficiencia: Si dos modelos tienen un rendimiento similar, se preferir estrictamente el que utilice menos variables.

1.3 Metodologa de Trabajo

El flujo de trabajo sigue el estndar de la industria para Ciencia de Datos:

  1. Auditora de Datos: Entender la naturaleza estadstica de las variables.
  2. Limpieza Preventiva: Deteccin de anomalas sin cometer Data Leakage.
  3. Seleccin de Caractersticas: Identificar qu variables realmente impactan el aprendizaje.
  4. Validacin Cruzada: Seleccionar el modelo por su estabilidad matemtica.
  5. Prueba Ciega: Validacin final con un conjunto de datos reservado (Test Set).

4. Seleccin de Caractersticas y Optimizacin del Modelo

Una vez que los datos estn limpios y transformados, el siguiente paso crtico en ingeniera de datos es la Parsimonia: encontrar el modelo ms simple posible que ofrezca la mayor precisin predictiva.

Introducir todas las variables disponibles sin criterio puede ocasionar:

  1. Sobreajuste (Overfitting): El modelo aprende el "ruido" en lugar de la tendencia.
  2. Complejidad Innecesaria: Ms variables implican mayor costo computacional y menor interpretabilidad.

Estrategia: Eliminacin Recursiva (RFE) Para seleccionar objetivamente las variables predictoras, utilizaremos el algoritmo RFECV (Recursive Feature Elimination with Cross-Validation).

  • Mecnica: Funciona como un torneo de eliminacin "hacia atrs" (Backward Elimination). Entrena el modelo con todas las variables, identifica la que menos aporta (menor coeficiente $\beta$), la expulsa y vuelve a entrenar.
  • Validacin: Este proceso se repite validando con 5 bloques de datos distintos (Cross-Validation) para asegurar que la seleccin del "equipo" de variables sea estadsticamente robusta.

  2. El Costo del Riesgo ($\beta_2 = -0.45$): El historial de materias reprobadas acta como un penalizador constante. Independientemente del promedio actual, tener antecedentes de fracaso reduce la expectativa de nota final en casi medio punto por materia.

  3. La Paradoja del Ausentismo ($\beta_3 = +0.08$): Un hallazgo contraintuitivo es la correlacin levemente positiva de las faltas.
